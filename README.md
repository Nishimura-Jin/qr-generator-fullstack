# 注釈付きQRコード生成ツール（フルスタック版）

## 概要

URLを入力するだけでQRコードを生成し、日本語の注釈付き画像としてダウンロードできるWebアプリです。

フロントエンド（React）とバックエンド（FastAPI）を分離し、API通信を通じてデータをやり取りする構成で開発しました。

履歴機能やバリデーション、エラー処理なども実装し、実用性とUXを意識したアプリケーションに仕上げています。

---

実際に操作できるURLです。

https://qr-generator-fullstack.vercel.app/

## Streamlit版との違い

以前にStreamlitでQRコード生成アプリを作成していました。

- Streamlit版: https://github.com/Nishimura-Jin/url-to-qr-generator

単一スクリプト構成であったため、UIの自由度やAPIとしての再利用性に課題がありました。

本プロジェクトでは、React + FastAPI の構成にすることでフロントエンドとバックエンドを分離し、より実務に近い設計に改善しました。

---

## 開発の背景

既存のStreamlit版では「動くもの」は作れましたが、UIの自由度に限界があり、またAPIとして外部から利用できないという課題がありました。

そこで、フロントエンドとバックエンドを分離した構成に変更し、APIベースでデータをやり取りする設計にしました。

さらにDockerを用いて開発環境を統一し、再現性のある構成でフルスタックアプリとして作り直しました。

---

## 機能一覧

- QRコード生成（URLまたは任意のテキスト）
- 日本語注釈の追加（QRコードの上または下に配置）
- 生成画像のPNGダウンロード
- リンク先をブラウザで開く機能
- 生成履歴の保存・一覧表示（クリックで再利用可能）
- 履歴の個別削除
- URL形式チェック（http / https）

---

## 使用技術

- **Language:** Python 3.11 / JavaScript
- **Frontend:** React 19 + Vite 8
- **Backend:** FastAPI
- **Libraries:**
  - qrcode（QRコード生成）
  - Pillow（画像処理・テキスト描画）
  - sqlite3（履歴管理）
- **Infrastructure:** Docker / Docker Compose
- **Dev Environment:** WSL2 (Ubuntu)

---

## 技術選定理由

### フロントエンド（React）

実務で広く使われている技術であり、コンポーネント単位でUIを管理する設計を学びたかったため採用しました。

---

### バックエンド（FastAPI）

フロントエンドとバックエンドを分離した構成を実現するため、APIを構築できるFastAPIを採用しました。

API化することでフロントエンドに依存せず、他のアプリケーションからも利用できる点も理由の一つです。

---

### Docker

環境構築の差異をなくし、コマンド一つで開発環境を再現できるようにするため導入しました。

---

## こだわった点

### 入力バリデーション

URLが空欄の場合や、`http / https` で始まらない場合にエラーを表示し、不正な入力の送信を防ぐようにしました。

---

### 状態管理の分離

入力中のURLと生成結果のURLを別々に管理することで、入力変更による表示の不整合を防ぎました。

---

### エラーハンドリング

API通信を共通化し、サーバーからのエラーメッセージを表示することで、原因が分かりやすいようにしました。

---

### UXの改善

QRコード生成後に結果表示部分へ自動スクロールする処理を追加し、ユーザーが結果を見つけやすいようにしました。

---

### フォント読み込みエラーへの対応

日本語フォントが利用できない環境でも処理が止まらないように、デフォルトフォントを使用するようにしました。

---

## 苦労した点

### 状態管理による不整合の発生

生成後に入力欄のURLを変更すると、表示・コピー・リンク先がすべて変わってしまう問題がありました。

入力用と生成結果用のstateを分離することで、生成後の内容が変わらないように改善しました。

---

### API連携時のエラー処理

エラー発生時に原因が分かりづらかったため、API通信処理を共通化し、メッセージを明示するようにしました。

---

### 日本語フォントの対応

環境によって日本語が正しく表示されない問題があったため、フォントを明示的に読み込み、失敗時の対策も実装しました。
